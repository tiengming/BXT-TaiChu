
# 卜仙堂 (BuXianTang) - 品牌门户：鸿蒙开辟篇

> 这是一个基于 Cloudflare Workers 部署的品牌导航单页，通过 3D 粒子引擎与 WebGL 动画，演绎从混沌开辟到金丹化境的东方哲学视觉。

## 🪐 视觉哲学 (Visionary Narrative)

本项目不仅是一个导航页，更是一场关于“道”的动态演示：
- **混沌期:** 暗色调 `漆黑 #161823` 代表未知的宇宙原点。
- **化生期:** `朱砂 #FF461F` (阳/天) 与 `玄青 #232021` (阴/地) 丝线交融。
- **开辟期:** 金丹碰撞引发“宇宙大爆炸”，视觉色彩从暗转明，背景切换至 `缟羽 #F1F1F1`。
- **呈现期:** `黄金 #EACD76` 丝线最终凝结为“卜仙堂”字样。
其具体动态演示过程为：
### 🎨 视觉叙事逻辑完善

1.  **第一阶段：混沌 (Primordial Chaos)**
    * **背景：** 采用【漆黑】(#161823) 渲染，加入极细微的、如星云般的暗紫色颗粒流动。
    * **氛围：** 视口中心有微弱的呼吸感，像是在黑暗中孕育着某种生命。

2.  **第二阶段：阴阳化生 (Duality of Yin & Yang)**
    * **动效：** 天降红丝：一缕【朱砂】(#FF461F) 色金丝带从屏幕顶端盘旋而下，路径轻盈。
        * 地升黑丝：一缕【玄青】(#232021) 色丝带（边缘带微弱光晕以区别背景）从底端向上蜿蜒。
    * **冲突感：** 两条线在接近中心时，周围空间开始扭曲（Glow 效果增强）。

3.  **第三阶段：金丹大成 & 宇宙大爆炸 (The Big Bang)**
    * **交汇点：** 红黑丝线在几何中心碰撞，瞬间化为一粒旋转的【黄金】(#EACD76) 金丹。
    * **爆炸转场：** 金丹骤然收缩后猛烈扩张，产生全屏白屏闪烁（模拟太初之光）。
    * **色调翻转：** 画面由【漆黑】瞬间转变为**【缟羽】(#F1F1F1)**（一种带有宣纸质感的柔和白色）。此时，黑色的墨韵在背景中炸开，如同在清水中滴入重墨。

4.  **第四阶段：勾勒真名 (The Manifestation)**
    * **Logo生成：** 爆炸留下的金色流光线段迅速重组，以书法笔触勾勒出“卜仙堂”三字。
    * **交互界面：** 文字下方，导航矩阵（博客、解析、社媒）以“印章”或“简约图标”的形式伴随着墨晕渐变浮现。

## 🛠️ 技术实现框架 (Technology Stack)

由于部署在 **Cloudflare Workers** 环境，项目采用 **Single-file Architecture** 或 **Static Assets Injection** 方案：

- **渲染引擎:** `Three.js` (处理金丝运动轨迹、粒子爆炸特效、颜色反转 Shader)。
- **动画控制:** `GSAP` (核心时间轴控制：从丝线接触到爆炸转场的高精度同步)。
- **排版引擎:** `HTML5 Canvas + Tailwind CSS` (负责极简导航组件的动态淡入)。
- **部署环境:** `Cloudflare Workers` + `KV / R2` (用于存储可能需要的材质贴图)。

## 🎞️ 动效细节规范

### 1. 丝线交融 (The Duality)
- **物理特性:** 红黑丝线需具备流体动力学效果，相互环绕但初始不接触。
- **色彩变化:** 接触瞬间，丝线交汇处色彩混合为 `#EACD76` (黄金)。

### 2. 爆炸反转 (The Big Bang)
- **触发器:** 当金丹旋转频率达到阈值。
- **视觉反馈:** - 粒子系统向外散射。
  - 背景 `backgroundColor` 在 200ms 内完成从 `#161823` 到 `#F1F1F1` 的突变。
  - 所有的金色线条增加 `Bloom` 滤镜，形成光芒万丈的效果。

### 3. 矩阵入口 (Navigation Matrix)
- **博客入口:** 采用【漆黑】色块线条，悬停变为【朱砂】。
- **经典解析:** 采用【漆黑】色块线条，悬停变为【黄金】。
- **社交媒体:** 采用【漆黑】色块线条，悬停变为【油烟墨】。

### 💡 针对 CF Worker 的特别建议：
由于 CF Worker 对脚本体积有一定限制，建议：
* **资源加载：** 核心字体文件和较大的贴图文件放在 **CF R2** 或 **Github Pages/CDN** 上，避免直接打包进 Worker 脚本。
* **库的引入：** 使用 **CDN 加载 Three.js 和 GSAP**，不仅能加快加载速度，还能保持 Worker 逻辑的纯净。
* **交互逻辑：** 在金丹化境完成后（即动画结束），再监听鼠标/点击事件，避免加载过程中的动画掉帧。
